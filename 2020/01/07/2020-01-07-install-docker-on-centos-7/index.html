常见问题

  1. 重启容器出现错误“Error response from daemon: Cannot restart container wp: oci runtime error: container with id exists: 547ecaa980a4be72120176fddc580c4fd3526a3c08831e016f996c01be36d846”

  2. 使用 docker update 修改运行中容器的重启策略

    你可以直接通过 docker update 命令为运行中的容器设置重启策略，而无需停止容器：

    其他可选的 --restart 策略包括：

    • no：不自动重启容器（默认值）。
    • on-failure[:最大重试次数]：当容器因错误退出（非零退出码）时重启，可以指定最大重试次数。
    • always：无论容器因何原因退出，始终重启。
    • unless-stopped：始终重启容器，除非手动停止。
