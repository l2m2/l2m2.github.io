Git Rebase

Dec 24, 2020 | Git | 阅读

通过一个例子来说明rebase的作用。

从字面上看来讲，rebase就是re+base，翻译过来大概是重新定义分支的参考基准。

base就是指这个分支是从哪里生出来的。

场景一：在过时的分支上开发, 日志分叉了

我们新建了一个工程dirty，只有一个README.md。

分别clone到本地不同的两个目录，以模拟多人协作。

A

B

A新建一个a.txt, push到远程。

然后B再新建一个b.txt, push到远程。

B push到远程时发生错误，提醒我们要先pull。

没啥问题，B来看看日志。

提交历史分叉了。不太舒服，这个时候我们可以使用rebase。

提交历史的分叉没有了。
